# 项目概览

## 当前状态

项目正在进行架构发现阶段。Stage 0 初步确认项目以 Python 数据工程为核心。Stage 1 已确认首版面向 VPS 定时运行和保守研究分析报告。Stage 4 前更新范围：需要增加轻量 Web UI 和数据库保存能力，用于快速查看分析报告和图表。

## 语言约定

- 交互语言：中文。
- 文档语言：中文。
- 代码注释语言：中文。

## 待确认

- 业务目标与最小可交付范围。
- 用户流程、领域模型、数据来源、外部依赖和测试交付策略。

## 初步项目定位

- 项目名称：日本宏观政策效果监控器。
- 核心用途：判断日本积极财政、弱日元和再通胀政策是否带来真实实际增长，还是主要推高名义 GDP 与通胀。
- 输出物：数据库中的结构化清洗数据、动态 Web 图表、可选 Markdown 归档。
- 输出界面：轻量 Web UI，用于查看最新报告、历史报告、核心指标和图表。
- 首选技术栈：Python 3.11+、pandas、requests、pyyaml、openpyxl、beautifulsoup4、FastAPI、Jinja2、SQLite、ECharts。
- 部署形态：VPS 上定期执行命令。
- 分析口径：保守研究分析，不输出投资建议。

## 数据源测试状态

- 已完成官方入口页和样例文件下载测试，记录见 `docs/source-automation-test.md`。
- GDP、e-Stat GDP、JGB 利率可直接自动化。
- CPI 和工资可自动化，但需要固定 e-Stat 筛选规则和目标表。
- USDJPY 长期序列仍需进一步确认 BOJ Time-Series Data Search 的稳定接口。

## 首版交付边界

- 自动下载、清洗、分析并生成报告。
- 提供统一入口 `python run.py`。
- 支持 cron 等 VPS 定时执行方式。
- Web UI 从数据库动态展示报告和 ECharts 图表。
- 面向经济基础较少的读者，报告需要结论先行、解释清楚、避免过度专业化。
- 将清洗后的结构化数据保存到数据库。
- Web UI 直接从数据库动态生成页面。
- Markdown 报告作为可选导出和归档格式，不作为 Web UI 的主要来源。

## 初步运行策略

- VPS 每天上午定时运行轻量检查。
- 关键数据源出现新数据后生成正式报告。
- 若发布日下载失败或官网未及时更新，当天晚间补跑。
- 最新报告应提供固定路径，例如 `outputs/reports/latest.md`。

## 判断框架

- 使用 0-100 分内部评分，并映射为易读标签。
- 核心关注真实增长、通胀压力和财政压力三类判断。
- 缺失关键数据时降低置信度，必要时只输出“数据不足，需继续观察”。

## Web UI

- 公开只读访问。
- 不做账号管理。
- 单一浅色报表样式。
- 首版页面：首页、历史报告、报告详情、数据源状态。
- 图表使用 ECharts 动态构建，不预生成 PNG。

## 成本控制

- 使用官方免费数据源。
- 使用 SQLite 本地数据库。
- 使用 VPS cron 定时执行，不引入额外任务队列服务。
- 首版不使用付费 API 和复杂前端框架。

## 交付要求

- 支持数据库初始化命令。
- 支持样例数据冒烟运行。
- 支持真实数据下载运行。
- 使用 Docker Compose 管理项目服务。
- 支持 VPS cron 调用 Docker Compose 批处理命令。
- Web UI 在空数据、部分失败和正常数据三种状态下都必须可用。
