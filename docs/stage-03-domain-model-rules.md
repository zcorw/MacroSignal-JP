# Stage 3. 领域模型与规则

## 已确认事实

- 目标读者是经济基础较少的使用者。
- 报告需要尽量易读，但底层判断要保持专业、保守和可追溯。
- 系统关注的是日本政策是否带来真实实际增长，而不是仅推高名义 GDP 和通胀。
- 项目首版采用自动化可行的数据源和规则框架，后续再逐步增强。

## 已确认决策

- 报告采用“结论先行 + 证据解释 + 风险提示”的结构。
- 评分采用 0-100 分，但对读者展示时同时映射为低/中/高或偏弱/中性/偏强。
- 不把单一指标当作最终结论，采用多指标证据框架。
- 缺失关键指标时，报告必须显著标注“数据不足”或“判断置信度下降”。
- 专业术语首次出现时需要用简短中文解释。

## 核心领域对象

### 数据源

- `GDP`：实际 GDP、名义 GDP、消费、投资、公共需求、进出口、GDP deflator。
- `Price`：CPI、核心 CPI、食品 CPI、能源 CPI。
- `WageConsumption`：名义工资、实际工资、消费相关指标。
- `MarketPressure`：USDJPY、JGB 10Y、20Y、30Y、BOJ 政策利率。

### 指标

- 原始指标：从官方 CSV、Excel、页面或手动文件读取。
- 标准化指标：统一日期、频率、单位和字段名后的数据。
- 派生指标：YoY、3 个月变化、差值和评分。

### 报告判断

- `real_growth_score`：真实增长评分。
- `inflation_pressure_score`：通胀压力评分。
- `fiscal_stress_score`：财政压力评分。
- `confidence_level`：判断置信度。
- `summary_label`：面向读者的简明判断标签。

## 推荐评分框架

### 真实增长评分 `real_growth_score`

满分 100，建议权重：

- 实际 GDP YoY 为正且改善：25 分。
- 实际工资 YoY 为正或明显改善：25 分。
- 实际民间消费 YoY 为正：20 分。
- 企业设备投资 YoY 为正：15 分。
- CPI 不再明显高于名义工资增长：10 分。
- JGB 长端收益率未快速上升：5 分。

解释：

- GDP 代表总体经济。
- 实际工资和消费代表居民是否真的更有购买力。
- 企业投资代表企业是否相信未来增长。
- 通胀和 JGB 用来检查增长是否被价格上涨或财政压力扭曲。

### 通胀压力评分 `inflation_pressure_score`

满分 100，分数越高表示通胀压力越大：

- GDP deflator YoY 高：20 分。
- CPI YoY 高：20 分。
- CPI 高于名义工资增长：20 分。
- 食品或能源 CPI 明显高：15 分。
- USDJPY 三个月贬值明显：15 分。
- 名义 GDP YoY 明显高于实际 GDP YoY：10 分。

### 财政压力评分 `fiscal_stress_score`

满分 100，分数越高表示财政和国债市场压力越大：

- JGB 10Y 三个月上升超过 50bp：30 分。
- JGB 30Y 三个月上升超过 75bp：30 分。
- 10Y/30Y 同时上行：20 分。
- USDJPY 同期明显贬值：10 分。
- 通胀压力评分较高：10 分。

## 初始阈值

- `JGB 10Y` 三个月上升超过 50bp：长端利率压力明显。
- `JGB 30Y` 三个月上升超过 75bp：超长期财政压力明显。
- `USDJPY` 三个月上升超过 5%：日元贬值压力明显。
- `nominal_gdp_yoy - real_gdp_yoy` 超过 2 个百分点：名义增长中通胀贡献偏高。
- `wage_minus_cpi < 0`：工资增速落后于 CPI，居民购买力承压。

这些阈值是首版保守规则，不是经济学定律。报告中应写明“规则判断”，避免伪装成确定结论。

## 面向读者的结论标签

- `真实增长证据增强`：实际 GDP、实际工资、消费、投资多数改善，通胀和 JGB 压力可控。
- `名义增长成分偏高`：名义 GDP 明显强于实际 GDP，GDP deflator 或 CPI 较高。
- `居民购买力仍承压`：实际工资为负，或工资增速低于 CPI。
- `通胀压力上升`：CPI、食品/能源、GDP deflator 或弱日元压力明显。
- `财政压力需警惕`：JGB 长端收益率快速上升，尤其 10Y/30Y 同时上行。
- `数据不足，需继续观察`：关键数据缺失、滞后或冲突。

## 缺失数据处理

- 缺失 GDP、CPI、工资任一核心数据时，整体结论置信度下降。
- 缺失两个以上核心模块时，不输出强结论，只输出“数据不足”。
- 市场数据缺失时，仍可分析真实增长和通胀，但财政压力判断必须降级。
- 手动导入数据参与报告时，必须标注文件名、更新时间和来源说明。

## 报告解释规则

- 每个结论必须给出 2-4 条关键证据。
- 每个评分必须给出“为什么加分/扣分”。
- 专业词汇首次出现需要简短解释，例如：
  - 实际 GDP：扣除价格上涨后的经济产出。
  - 名义 GDP：没有扣除价格上涨的经济总量。
  - GDP deflator：衡量 GDP 内部价格变化的指标。
  - 实际工资：扣除通胀后的工资购买力。
  - JGB：日本国债。
- 报告避免直接说“政策成功/失败”，优先说“当前数据更支持/不支持某种解释”。

## 后续可调整项

- 阈值和权重可以在 `config.yaml` 中配置。
- 如果未来读者更专业，可以增加详细附录。
- 如果未来需要投资决策版本，应另建更严格的风险声明和市场数据模块。
