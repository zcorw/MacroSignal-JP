# 项目约定

## 语言

- 面向维护者的文档使用中文。
- 自有代码注释默认使用中文。
- 公共 API 字段、配置键、数据库字段和代码标识符使用英文 snake_case。

## 分析口径

- 项目输出采用保守研究分析口径。
- 报告必须区分事实数据、派生指标、规则判断和数据缺口。
- 面向读者的结论必须使用保守措辞，不把规则评分写成确定事实。
- 专业术语首次出现时应给出一句话解释。
- 每个核心判断必须附带关键证据和数据缺口说明。

## 数据管道

- 定时任务必须保留日志。
- 失败时不得覆盖上一次成功报告。
- 数据源更新检测应基于数据日期、文件元信息或内容哈希。
- 若使用手动数据兜底，必须标注来源和更新时间。
- 清洗后的结构化数据必须入库。

## Web UI

- 首版 Web UI 为公开只读页面。
- 不实现账号管理、用户权限和后台管理。
- 页面不得暴露服务器绝对路径、环境变量、密钥或异常堆栈。
- 失败信息在 UI 中只展示可读摘要，详细错误写入日志。
- Web 层不得调用下载、清洗、分析函数。
- Web UI 只读数据库，不提供写操作接口。
- 图表必须基于数据库数据用 ECharts 动态绘制，不预生成 PNG 作为主展示。

## 依赖与成本

- 首版不使用付费 API。
- 首版不引入 React、Next.js、Redis、Celery、PostgreSQL 等额外服务。
- 除官方数据下载外，不依赖外部网络服务生成报告。
- 所有外部 URL 必须集中配置或集中定义。
- 新增依赖必须有明确收益。

## 数据与 API

- 序列名和指标名使用英文 snake_case。
- UI 展示必须映射为中文名称。
- 所有日期使用 ISO 格式。
- JSON API 只读，不提供写接口。
- ECharts 图表 API 返回结构必须稳定。
- 手动数据文件必须包含来源字段。

## Docker Compose 与交付

- 项目使用 Docker Compose 管理运行服务。
- `data/` 和 `logs/` 必须作为持久化目录挂载。
- VPS cron 调用 Docker Compose 批处理命令。
- 生产部署不直接对公网暴露 uvicorn，应通过 Nginx 代理。
- cron 日志必须写入 `logs/`。
- Web UI 在数据库为空时必须显示空状态。
- 联网测试和单元测试分开。
