# Stage 1. 业务目标与范围

## 已确认事实

- 项目目标是“日本宏观政策效果监控器”。
- 系统将部署在 VPS 上运行。
- 系统通过定期执行命令完成自动下载、清洗、分析和报告生成。
- 输出需要便于阅读和理解，面向研究分析场景，而不是只输出数据表或图表。
- 分析口径采用“保守研究分析”，避免给出过度确定的投资或政策结论。
- 数据源自动化测试已完成，结论见 `docs/source-automation-test.md`。

## 已确认决策

- 首版交付形态：Python 数据管道 + 定时运行能力 + 结构化数据库 + 轻量 Web UI + ECharts 动态图表 + 可选 Markdown 归档。
- Web UI 用于快速查看最新报告、历史报告和图表，不做复杂交互式仪表盘。
- 官方数据源自动化优先；对于暂未稳定自动化的数据源，允许提供手动文件导入兜底。
- 判断框架采用保守研究口径：
  - 用“证据支持/证据不足/存在压力/需继续观察”等表达。
  - 不输出交易建议。
  - 对缺失数据、滞后数据和口径变更明确标注。

## 首版范围

- 自动下载可稳定获取的数据：
  - ESRI GDP CSV。
  - e-Stat GDP CSV。
  - e-Stat CPI Excel。
  - e-Stat Monthly Labour Survey Excel。
  - 财务省 JGB 利率 CSV。
  - BOJ 可稳定下载的 flat file 或页面数据。
- 自动清洗并保存标准化数据到 `data/processed/`。
- 计算核心派生指标，包括实际/名义 GDP YoY、GDP deflator YoY、工资 YoY、CPI YoY、工资-CPI 差、消费与投资 YoY、JGB 和 USDJPY 的阶段变化。
- 生成结构化分析结果，Web UI 动态展示报告和图表。
- 提供统一入口 `python run.py`。
- 提供 VPS 定时执行说明，优先支持 cron。

## 暂不纳入首版范围

- 用户登录、多用户权限、数据库后台。
- 实时行情系统。
- 投资建议或交易信号。
- 对 BOJ Time-Series Data Search 的复杂浏览器自动化，除非后续确认无更稳接口。

## 风险与缺口

- USDJPY 长期序列仍需确认官方可自动化路径；首版可能需要 BOJ 近 70 个营业日数据或手动 CSV 兜底。
- e-Stat CPI 和工资数据需要固定目标表，避免数据表随页面排序变化。
- VPS 定时任务需要考虑失败重试、日志留存、输出归档和数据源临时不可用。
- Web UI 的具体技术栈、页面范围和访问控制仍待确认。

Stage 2 将确认用户流程与运行场景，包括 VPS 上的定时频率、报告阅读方式、失败通知和人工复核流程。
